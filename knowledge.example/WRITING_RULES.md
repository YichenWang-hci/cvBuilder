# 资料库写作规则（全职实习 · 禁止套话）

## 简历 8 段结构

| § | 区块 | 可变 | 文件 |
|---|------|------|------|
| 1 | 姓名 | 否 | `profile.yaml` → `name` |
| 2 | 自我描述 | 是 | `profile.yaml` → `summaries` |
| 3 | 学习经历 | 是 | `profile.yaml` → `education` |
| 4 | 项目经历 | 是 | `projects/*.yaml` |
| 5 | 联系方式 | 否 | `profile.yaml` → `contact` |
| 6 | 个人能力 | 是 | `abilities.yaml` |
| 7 | 软件能力 | 是 | `software.yaml` |
| 8 | 语言能力 | 否 | `profile.yaml` → `languages` |

Figma 出稿：`templates/resume-schema.json` + `templates/figma-template-map.yaml`；填充见 `fill_figma.py` / `figma-plugin/`。Agent 规则见 `agent/prompts/generate.md`。

**固定信息与日期格式**：见 `templates/DATA_FORMAT.md`（德式日月年、缺失字段策略）。录入脚本：`onboard.py`。

## 日期（德式 · 全库统一）

| 场景 | 格式 | 示例 |
|------|------|------|
| 教育/项目起止 | `开始 - 结束` | `01.04.2025 - heute` |
| 仅精确到月 | `MM.JJJJ` | `04.2025 - 07.2025` |
| 德语月份名 | `Monat JJJJ` | `Oktober 2023 - Oktober 2024` |
| 进行中 | 结束写 `heute` | `April 2025 - heute` |

**禁止**在资料库存 ISO：`2025-04-01`、`2025-10`（录入时自动转换或警告）。

`period` 为空 → 保留在 yaml，生成简历时跳过该条，末尾报告提醒补全。

## 双语约定

**存储双语，输出单语。**

| 阶段 | 语言 |
|------|------|
| 资料库（yaml） | 每条文案写 `{ de, en }` 两种 |
| **单次生成（resume.json）** | **只用一种**：`de` **或** `en` |

```yaml
# 资料库 — 双语存储
text:
  de: "德语文案"
  en: "English text"
```

```json
// 生成结果 — 单语输出（示例：output_language = de）
{ "value": "德语文案" }
```

- **姓名、邮箱、电话、地址**：语言无关，只写一份
- **学校名**：通常只写一份；**专业名** `degree` 写 de/en
- **项目 title、role、bullet、summary、abilities**：资料库必须 de + en；生成时只取一侧
- **软件名**（Figma、Python）：通常 de/en 相同，仍用 `{ de, en }` 保持结构统一
- **§8 语言**：资料库 `name` / `level` 各写 de/en；生成时整段用同一语言侧

**禁止**：同一份 `resume.json` 中德英混排；禁止把 `{ de, en }` 对象直接写入输出。

需要两种语言版本 → 运行**两次**生成，分别指定 `output_language: de` 与 `output_language: en`。

## 目标

找 **全职实习**：能全职到岗、能独立交付小模块、能在 8–12 周内产生可验证产出。
资料库是所有简历/动机信的 **唯一事实来源**。Agent 只能改写，不能编造。

---

## 禁止出现的套话（简历 & 动机信）

以下词汇/句式 **不得单独出现**，若出现必须附带具体证据（项目名、数字、工具、交付物）：

- 热爱 / 充满热情 / passionate about
- 沟通能力强 / 团队合作 / team player
- 学习能力强 / fast learner / 快速上手
- 责任心强 / 抗压能力强
- 对 XX 行业有深入了解（没有具体接触经历时）
- 贵公司行业领先 / 久负盛名（空洞恭维）
- 我相信我能为公司带来价值（无具体内容）
- 综合能力强 / 全方位发展

**替换方式**：不说「学习能力强」，说「两周内用 FastAPI 重写原 Flask 接口，测试覆盖从 0 到 62%」。

## 每条 bullet 必须满足

```
[动词] + [做了什么] + [用什么] + [结果/可验证产出]
```

| 合格 | 不合格 |
|------|--------|
| 用 Redis 缓存热点查询，P95 从 800ms 降到 120ms | 优化系统性能，效果显著 |
| 交付可运行的 MVP，含登录、列表、导出 3 个页面 | 参与核心功能开发 |
| 修复 12 个生产 bug，其中 3 个为支付链路 | 负责 bug 修复与维护 |

**数字**：有就写真实数字；没有就写可观察结果（「上线后同学日活使用」「导师用于课程 demo」），不要编。

## 全职实习简历策略

1. **一页纸**，最多 2–3 段经历（项目/实习/课程设计），每段 2–3 条 bullet
2. **技能**：只列 JD 相关 + 你有项目背书的；不要堆 20 个关键词
3. **Summary（若有）**：1–2 句，写「你能做什么 + 何时可全职到岗」，不写人生理想
4. **教育**：学校、专业、预计毕业时间；GPA 仅在有竞争力时写
5. **避免**：「主导」「带领团队」除非真有（课程组长、实习 mentor 反馈可写）

## 动机信结构（250–350 词，英文或中文按 JD）

1. **第一段（3–4 句）**：为什么是这个 **岗位**（引用 JD 里一个具体职责，说明你已经做过类似的事）
2. **第二段（4–5 句）**：一个 **最相关项目** 的 concrete story（问题 → 你的动作 → 结果）
3. **第三段（2–3 句）**：**全职实习** 可用性（何时到岗、每周几天、持续多久）+ 一个你对该公司 **具体产品/业务** 的观察（来自官网/JD/新闻，非空泛夸奖）

**禁止**：开头「尊敬的 HR」后接一整段自我吹嘘；结尾「期待您的回复」单独占段。

## Markdown 项目文档写什么

给 Agent 「讲故事」用，不是给 HR 直接看：

- 真实背景（谁的需求、时间压力、资源限制）
- 你做过的 **具体决策** 及原因（为什么选 A 不选 B）
- 搞砸或妥协的地方（动机信可以写 growth，但要真实）
- 若再实习一次，你会怎么做得不一样

---

## 给 Agent 的硬性约束

生成内容时 Agent 必须：

1. 每条 bullet 标注来源 `project_id` + `bullet_id`
2. 未在资料库出现的能力 **不得写入**
3. 动机信中至少 **1 处引用 JD 原文短语**，**1 处引用资料库具体事实**
4. 输出后自检：若删掉所有公司名仍像通用模板 → 重写
